成为Javascript武士的六个秘笈

原文地址：http://www.infoworld.com/d/application-development/6-secrets-of-javascript-jedis-231322

作为一名前端开发者，我见过很多使用JavaScript编程的人。他们当中的大部分人仅仅使用jQuery和一些插件来制作一个奇特的灯箱效果或者一个幻灯片。与之相反，还有很多通过可以称之为武士级别的开发者，他们亲力亲为，在键盘上敲击代码来完成自己所需要的一切。
如果你正处于这两种级别之间，会常常不知道如何能够升级。进阶之路有很多，如果你正停滞不前并且愿意被推一把，那么这里就有几个JavaScript专家应该知道的知识点可以帮到你。

No1. 闭包

JavaScript的闭包允许一个函数访问它的父域。这是一个很容易让开发者迷惑的概念。在我学习JavaScript时，我花了大量时间去调试那些没有缩进的闭包代码。当我从错误中学习之后，我觉得闭包非常酷，所以我又白白花了更多的时间尝试使用闭包来解决自己的问题。
最终我认识到：当你需要返回一个内部函数 (参看No2. 高阶函数)，而且需要在这个函数里访问父域的时候，闭包就非常有用。这种方法为变量使用创造了一种完全私有或者受保护的环境。

// **Closures**

function makeThreeTimer() {

    var count = 0;

    return function() {

        if (count < 3) {

            console.log('doing work');

            count++;

        } else {

            throw new Error('No more work');

        }

    }

}

var threeTimer = makeThreeTimer();

threeTimer(); // logs 'doing work' (count gets incremented)

threeTimer(); // logs 'doing work' (count gets incremented)

threeTimer(); // logs 'doing work' (count gets incremented)

threeTimer(); // throws an error

threeTimer.count; // returns undefined

在上面的例子里，变量count在函数threeTimer被执行的时候——通过闭包函数——被使用和递增，而不能够直接通过点语法或者括号来访问。

No2. 高阶函数

在函数式编程语言里，函数是一等公民。它们可以像变量一样被来回传递，由此产生了很多有趣的可能性。
高阶函数是指那些使用其他函数作为输入或者输出另一个函数的函数。在Function.prototype里面的一些方法，例如：call和bind，就是高阶函数。类似函数currying和递归缓存（memoization）就都是采用高阶函数的形式来表达的。此外，JavaScript面向方面编程也可以通过高阶函数来实现。

No3. 调用函数

在理解了普通的函数调用（通过括号操作符）后，就该学习如何使用call和apply。使用call/apply而不是使用括号操作符来调用函数的便利之处在于，你可以指定函数执行的上下文（也就是this的值）。在这里你会经常看到高阶函数，尤其是使用其他函数作为输入用于以后执行的情况。Function.prototype内部的bind方法就是一个典型的call/apply的例子。

// Possible implementation of bind using apply

function bind(func, context) {

    return function() {

        func.apply(context, Array.prototype.slice.apply(arguments));

    }

}

No4. this是什么？

this这个关键字对很多JavaScript开发者而言都是个巨大的障碍，甚至完全避免使用它。我所见过的对这个问题最好的描述来自Yehuda Katz的关于函数执行的博客。在不使用call/apply或者bind的时候，this的值永远都是全局对象，除了下面的几个场景： 1. 问题函数是通过一个new操作符调用的，这时this的值就是这个新建的对象 2. 问题函数是某个对象的方法，这时this的值就是这个对象 规则2在相关函数被异步调用的时候应该被忽视，例如一个点击事件处理或者setTimeout。例如：

Person.getName(); // 'this' points to Person

setTimeout(Person.getName, 1000); // 'this' points to the global object

No5. 保护全局域

JavaScript的瑕疵之一就是在一个页面内所有的脚本都在一个共享的全局上下文里执行。使用不当就会导致站点易受攻击，例如跨站脚本攻击。共享的全局上下文还会导致其他问题。例如，很多脚本运行在一个页面上，但是并不是所有的脚本都能够被你识别，比如广告。这些脚本在全局空间里运行，就能够访问相同的全局变量。如果两个脚本正好使用了相同的全局变量，它们就开始互相干扰，代码因此崩溃。
最小化全局域的直接使用是有助于排除代码互相干扰，节省你的查错时间的一种保护性技术。
虽然不能消除存储全局变量，但是可以使用命名空间技术来最小化这种影响：

// **Namespacing**

// The global footprint is a single object containing

// all other references needed.



var MyApp = {};



MyApp.id = 42;

MyApp.utils = {

    validate: function() {

        //do work

    }

};

或者使用模块化技术：

// **Module pattern**

// This relies on immediately invoked anonymous functions and the

// closure they create to maintain access to variables within the

// function.



var MyApp = (function() {

    //declare a local variable

    var appId = 42;



    function getAppId() {

        //referring to a variable in this function's parent scope

        return appId;

    }



    return {

        getAppId: getAppId

    };

}());

appId; // undefined

MyApp.appId; //undefined

MyApp.getAppId(); // returns 42. (Brought to you by closures)

No6. 继承

因为多种原因，在对象继承方面，JavaScript有一段冗长而混乱的历史。大部分开发者——包括许多我认识的JavaScript开发者——都对经典模型有一种固定的理解从而完全搞不清楚原型模型。精研一些采用原型模型继承的语言以后应该会增强认识。
JavaScript的原型模型继承不同于任何主流语言，更糟糕的是，JavaScript也可以模拟经典模型继承。因此就产生了各种继承的方式，其中不乏自以为是和自相矛盾的。我建议完全规避这种伪经典模型的办法，因为这是JavaScript的警示之一。对大部分开发者而言貌似熟悉，但实际上这种方式会很快崩溃，因为它根本就只是一个赝品。
